This marks a very important moment for any project that follows <a href="https://semver.org/">semver</a> as we are finalizing our public API (no more breaking changes for a while) and enter a <a href="https://github.com/fastify/fastify/blob/master/docs/LTS.md">Long Term Support</a> cycle. We are committed to keeping 1.0.0 stable and safe for at least a year (6 months of active support + 6 months of security fixes).</p><p>Fastify is very simple to use and install, simply type`npm i fastify` and then:</p><a href="https://medium.com/media/89f0e9865b391c9c3510221ffbf5cfa3/href">https://medium.com/media/89f0e9865b391c9c3510221ffbf5cfa3/href</a><h3>Our story</h3><p>It has been a long journey beginning in the summer of 2016, when we started developing <a href="https://github.com/fastify/fast-json-stringify">fast-json-stringify</a>, a little library that doubled the throughput of `JSON.stringify`. In September 2016, Fastify was born. We initially focused only on performance, but after the first iterations, we realized that performance is not the only important aspect that a framework should offer. So, we started thinking about a way to empower developer productivity and extend the framework itself. By offering the possibility to add plugins and interact with the framework API and lifecycle directly, we enhanced the developer experience.</p><p>While we were continuously improving and optimizing, the stability and resiliency of the API was our main focus. This iterative process finally came to fruition when we perfected the performance and the overall usability of the framework. Fastify 1.0.0 was finally ready to be launched.</p><h3>Features</h3><p>Fastify offers:</p><ul><li>the familiar “callback style” API for die-hard Node.js developers;</li><li>async/await, because we like ES6 too;</li><li>HTTP2 support;</li><li>a powerful hook system that allows the developer to interact with the lifecycle of the application with a minimum overhead;</li><li>support of Express-style middlewares;</li><li>validation of incoming data with JSON-schema;</li><li>zero-cost decorators to enhance the base API;</li><li>a <a href="https://github.com/delvedor/find-my-way">router</a> based on a radix prefix tree, built with performance in mind;</li><li>a built-in <a href="https://github.com/pinojs/pino">logger</a>, because full auditability is a critical feature in enterprise applications (and it’s very easy to underestimate <a href="https://vimeo.com/180476148">the cost of logging</a>);</li><li>a flexible plugin system based on a <a href="https://en.wikipedia.org/wiki/Directed_acyclic_graph">direct acyclic graph</a>, based on <a href="https://github.com/mcollina/avvio">Avvio</a>, which enables our awesome ecosystem;</li></ul><p>There are more than <a href="https://www.fastify.io/ecosystem/">50 plugins</a> that have been developed, including those for template rendering, React integration, GraphQL support, serving static files, and database drivers.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*y2TdQ2AccIPky9_56Qx5qQ.jpeg" /><figcaption>Courtesy of Roberta Doyle <a href="https://unsplash.com/photos/dqmmx51pVf8">https://unsplash.com/photos/dqmmx51pVf8</a></figcaption></figure><h3>Open Source Impact</h3><p>Open Source is in our hearts. We try to release every feature as a separate package to enable others to use them in their project. That’s how we believe the project and its ecosystem can organically evolve.</p><p>As part of our investment to share the knowledge we gained during the development, we focused on compiling an exhaustive documentation, including our technical decisions.</p><p>As a result we have a comprehensive guide to plugins in our <a href="https://github.com/fastify/fastify/blob/master/docs/Plugins-Guide.md">Hitchhiker’s Guide to Plugins</a>, a thorough introduction to the framework in our <a href="https://github.com/fastify/fastify/blob/master/docs/Getting-Started.md">Getting Started Guide</a>, and we recently published some <a href="https://github.com/fastify/fastify/blob/master/docs/Write-Plugin.md">suggestions</a> on how to create quality plugins for the ecosystem. For this reason, we strongly encourage third party developers to share their efforts to help continuously improving the ecosystem.</p><h3>Collaborate, collaborate, collaborate!</h3><p>During the past year a lot of amazing people joined our <a href="https://github.com/fastify/fastify#team">team</a>, and we would like to thank them: <a href="https://twitter.com/allevitommaso">Tommaso Allevi</a>, <a href="https://twitter.com/dustindeus">Dustin Deus</a>, <a href="https://twitter.com/loige">Luciano Mammino</a>, <a href="https://twitter.com/evanshortiss">Evan Shortiss</a>, <a href="https://twitter.com/jsumners79">James Sumners</a> and <a href="https://twitter.com/NathanWoltman">Nathan Woltman</a>.</p><p>Each one of them contributed to the development and the delivery of what we are all proud today to announce: Fastify 1.0.0. This was a cooperative effort done together with all the other <a href="https://github.com/fastify/fastify/graphs/contributors">contributors</a> that assisted us through this journey. We would also love to thank our sponsors, <a href="http://www.letzdoitapp.com">LetzDoIt</a> and <a href="https://www.nearform.com">nearForm</a> that enabled Tomas and Matteo to work on Fastify as part of their daily job.</p><p>So welcome everyone and enjoy Fastify v1.0.0!</p><p>Related post and talks:</p><ul><li><a href="https://thenewstack.io/introducing-fastify-speedy-node-js-web-framework/">https://thenewstack.io/introducing-fastify-speedy-node-js-web-framework/</a></li><li><a href="https://survivejs.com/blog/fastify-interview/">https://survivejs.com/blog/fastify-interview/</a></li><li><a href="https://www.youtube.com/watch?v=5z46jJZNe8k">Take your HTTP server to ludicrous speed</a></li><li><a href="https://www.webexpo.net/prague2017/talk/what-if-i-told-you-that-http-can-be-fast/">What if I told you that HTTP can be fast</a></li></ul><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=911112c64752" width="1" /><hr /><p><a href="https://medium.com/the-node-js-collection/fastify-goes-lts-with-1-0-0-911112c64752">Fastify goes LTS with 1.0.0!</a> was originally published in <a href="https://medium.com/the-node-js-collection">Node.js Collection</a> on Medium, where people are continuing the conversation by highlighting and responding to this story.</p>
