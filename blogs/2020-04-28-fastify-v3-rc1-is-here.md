<figure><img alt="cheetah" src="https://cdn-images-1.medium.com/max/1000/0*l2TaSFpfN9YXxqTW" /><figcaption>Courtesy of <a href="https://unsplash.com/@bird_mike">Mike Bird</a></figcaption></figure><p>After a long development cycle, we are enormously happy to announce the first release candidate of Fastify v3!</p><p>This new version of Fastify includes some changes that will allow you to have an even better experience with Fastify, starting from the brand new type definitions, the new middlewares support, and the refactored request validation!</p><p>Starting today, Fastify v3 will be released as Release Candidate, so you can try it and tell us if there is still something we must fix or improve before cut the final release. During the RC period, we will also update all of our official plugins to support at best Fastify v3. If you would like to help us out, check out the <a href="https://github.com/fastify/fastify/issues/2217">issue</a> where we are tracking all the work!</p><p>Fastify can now be installed with npm install fastify@next. All the updated plugins are being released with the @next tag as well, e.g. npm install fastify-jwt@next.</p><p>You can find the list of all the major changes we did, with the link to their pull request, a brief description, and a code examples in our <a href="https://github.com/fastify/fastify/blob/master/docs/Migration-Guide-V3.md">migration guide</a> document.</p><p>Let us know about any compatibility and upgrade issues at <a href="https://github.com/fastify/help">fastify/help</a>.</p><p>Happy coding!</p><p><em>The Fastify Team</em></p><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=e804fe18cb0e" width="1" />
